Was ist AWS Lambda?

PDF

AWS Lambda ist ein Datenverarbeitungsservice, mit dem Sie Code ausführen können, ohne Server bereitstellen oder verwalten zu müssen. AWS Lambda führt Ihren Code nur bei Bedarf aus und skaliert automatisch – von einigen Anfragen pro Tag bis zu Tausenden pro Sekunde. Sie zahlen nur für die genutzte Datenverarbeitungszeit. Wenn Ihr Code nicht ausgeführt wird, wird auch nichts berechnet. Mit AWS Lambdakönnen Sie Code für praktisch jeden Anwendungstyp oder Backend-Service ohne Verwaltungsaufwand ausführen. AWS Lambda führt Ihren Code auf einer hochverfügbaren Datenverarbeitungsinfrastruktur aus und erledigt die gesamte Administration der Datenverarbeitungsressourcen, einschließlich der Server- und Betriebssystemwartung, Kapazitätsbereitstellung, automatischen Skalierung sowie der Code-Überwachung und -Protokollierung. Sie müssen lediglich Ihren Code in einer der von AWS Lambda unterstützten Sprachen angeben.

Sie können AWS Lambda verwenden, um Ihren Code als Reaktion auf Ereignisse auszuführen, wie Änderungen an Daten in einem Amazon S3-Bucket oder einer Amazon DynamoDB-Tabelle, um Ihren Code in Reaktion auf HTTP-Anfragen mit Amazon API Gateway auszuführen oder Ihren Code mit API-Aufrufen unter Verwendung von AWS-SDKs aufzurufen. Dank dieser Funktionen können Sie Lambda verwenden, um auf einfache Weise Datenverarbeitungsauslöser für AWS-Services wie Amazon S3 und Amazon DynamoDB zu erstellen, in Kinesis gespeicherte Streaming-Daten zu verarbeiten oder ein eigenes Backend zu erstellen, das in AWS-Größenordnungen sowie mit AWS-Leistung und -Sicherheit arbeitet.

Sie können auch serverlose Anwendungen erstellen, bestehend aus Funktionen, die von Ereignissen ausgelöst werden, und sie automatisch mit CodePipeline und AWS CodeBuild bereitstellen. Weitere Informationen finden Sie unter AWS Lambda-Anwendungen.

Wann sollte ich AWS Lambda verwenden?

AWS Lambda ist eine ideale Datenverarbeitungsplattform für viele Anwendungsszenarios, vorausgesetzt, Sie können Ihren Anwendungscode in Sprachen erstellen, die von AWS Lambda unterstützt werden, und in der AWS Lambda-Standardlaufzeitumgebung sowie mit den von Lambda bereitgestellten Ressourcen ausführen.

Wenn Sie AWS Lambda verwenden, sind Sie nur für Ihren Code verantwortlich. AWS Lambda verwaltet die Datenverarbeitungsflotte, die eine Kombination aus Speicher, CPU, Netzwerk und anderen Ressourcen bietet. Dies sorgt für weniger Flexibilität, was bedeutet, dass Sie sich nicht bei Datenverarbeitungs-Instances anmelden oder das Betriebssystem für bereitgestellte Laufzeiten anpassen können. Diese Einschränkungen ermöglichen es AWS Lambda, operative und administrative Aktivitäten für Sie durchzuführen, einschließlich Bereitstellen von Kapazität, Überwachen des Flottenzustands, Anwenden von Sicherheits-Patches, Bereitstellen Ihres Codes und Überwachen und Protokollieren Ihrer Lambda-Funktionen.

Wenn Sie Ihre eigenen Datenverarbeitungsressourcen verwalten müssen, bietet Amazon Web Services auch andere Datenverarbeitungsservices, um Ihre Anforderungen zu erfüllen.

  • Der Amazon Elastic Compute Cloud (Amazon EC2)-Service bietet Flexibilität und eine breite Palette an EC2-Instance-Typen, aus denen Sie wählen können. Sie haben die Möglichkeit, Betriebssysteme, Netzwerk und Sicherheitseinstellungen sowie den gesamten Softwarestack anzupassen, Sie sind jedoch für das Bereitstellen der Kapazität, Überwachen des Zustands und der Performance der Flotte und das Verwenden von Availability Zones für Fehlertoleranz verantwortlich.

  • Elastic Beanstalk bietet einen benutzerfreundlichen Service zum Bereitstellen und Skalieren von Anwendungen auf Amazon EC2. Dabei bleiben Sie Eigentümer und behalten die vollständige Kontrolle über die zugrundeliegenden EC2-Instances.

Lambda ist ein Dienst mit hoher Verfügbarkeit. Weitere Informationen finden Sie unter AWS Lambda Service Level Agreement.

Verwenden Sie AWS Lambda zum ersten Mal?

Wenn Sie AWS Lambda zum ersten Mal verwenden, empfehlen wir Ihnen, nacheinander die folgenden Abschnitte zu lesen:

  1. Lesen Sie die Produktübersicht und sehen Sie sich das Einführungsvideo an, um Beispiel-Anwendungsfälle zu verstehen. Diese Ressourcen sind auf der AWS Lambda-Webseite verfügbar.

  2. Absolvieren Sie die konsolenbasierte Einstiegsübung. Die Übung bietet Anleitungen zum Erstellen und Testen Ihrer ersten Lambda-Funktion mit der Konsole. Außerdem erfahren Sie mehr über das Programmiermodell und andere Lambda-Konzepte. Weitere Informationen finden Sie unter Erste Schritte mit AWS Lambda.

  3. Lesen Sie den Abschnitt Bereitstellen von Anwendungen mit AWS Lambda in diesem Handbuch. In diesem Abschnitt werden verschiedene AWS Lambda-Komponenten zum Erstellen einer durchgehenden Erfahrung vorgestellt.

Neben der Einstiegsübung können Sie verschiedene Anwendungsfälle untersuchen, wobei für jeden ein Tutorial vorhanden ist, das Sie durch ein Beispielszenario leitet. Je nach Ihren Anwendungsanforderungen (beispielsweise, ob Sie einen ereignisgesteuerten Lambda-Funktionsaufruf oder einen bedarfsgesteuerten Aufruf wünschen) können Sie spezielle Tutorials durchgehen, die Ihre Anforderungen erfüllen. Weitere Informationen finden Sie unter Verwenden von AWS Lambda mit sonstigen Services.

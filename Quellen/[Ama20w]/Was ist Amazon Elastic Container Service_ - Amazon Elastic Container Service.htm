Was ist Amazon Elastic Container Service?

PDF
RSS

Amazon Elastic Container Service (Amazon ECS) ist ein hochskalierbarer, schneller Container-Management-Service, der einfach auszuführen, zu stoppen und zu verwalten. Ihre Container sind in einer Aufgabe definiert Definition, die Sie verwenden, um einzelne Aufgaben oder als Service auszuführen. Sie können Ihre Aufgaben und Services auf einer serverlosen Infrastruktur, die von AWS Fargate oder für weitere Kontrolle über Ihre Infrastruktur können Sie Ihre Aufgaben und Services auf einem Cluster von Amazon EC2 Instances, die Sie verwalten.

Amazon ECS ermöglicht es Ihnen, Ihre Container-basierten Anwendungen mit einer einfachen API zu starten und zu stoppen ruft an. Sie können den Status Ihres Clusters auch von einem zentralen Service abrufen und für bestehende Benutzer von Amazon EC2, greifen Sie auf viele bekannte Amazon EC2 -Funktionen.

Sie können die Platzierung Ihrer Container in Ihrem Cluster basierend auf Ihrem Ressourcenanforderungen, Isolationsrichtlinien und Verfügbarkeitsanforderungen. Mit Amazon ECS, Sie haben keine müssen Sie Ihre eigenen Cluster-Management- und Konfigurationsverwaltungssysteme betreiben oder sich Sorgen machen über die Skalierung Ihrer Management-Infrastruktur.

Amazon ECS kann verwendet werden, um eine konsistente Bereitstellung und Build-Erfahrung zu schaffen, zu verwalten und Batch- und Extract-Transform-Load-Workloads (ETL) skalieren und anspruchsvolle Anwendungen erstellen Architekturen auf einem Microservices-Modell. Weitere Informationen über Amazon ECS Anwendungsfälle und Szenarien, siehe Container-Anwendungsfälle.

Die Schaltfläche AWS Das -Container-Services-Team unterhält eine öffentliche Roadmap auf GitHub. Der Fahrplan enthält Informationen darüber, woran die Teams arbeiten und ermöglicht AWS Kunden zu erreichen, direktes Feedback geben. Weitere Informationen finden Sie unter AWS Container-Fahrplan.

Eigenschaften von Amazon ECS

Amazon ECS ist ein regionaler Service, der die Ausführung von Containern in einem hochverfügbaren über mehrere Availability Zones innerhalb einer Region hinweg. Sie können Amazon ECS -Cluster innerhalb einer neuen oder vorhandenen VPC. Nachdem ein Cluster gestartet und ausgeführt wurde, können Sie eine Aufgabe erstellen Definitionen, die definieren, welche Container-Images über Ihre Cluster ausgeführt werden sollen. Ihre Aufgabe -Definitionen werden verwendet, um Aufgaben auszuführen oder -Services zu erstellen. Container-Images werden in und gespeichert aus Containerregistern entnommen werden, z. B. Amazon Elastic Container Registry.

Das folgende Diagramm zeigt die Architektur eines Amazon ECS -Umgebung ausgeführt auf AWS Fargate.

In den folgenden Abschnitten werden diese einzelnen Elemente der Amazon ECS Architektur und näher zu beschreiben.

Container und Bilder

So stellen Sie Anwendungen auf bereit Amazon ECS, müssen Ihre Anwendungskomponenten zum Ausführen in Behälter. Ein Behälter ist eine standardisierte Einheit von Softwareentwicklung, die alles enthält, was Ihre Softwareanwendung benötigt, um Ausführen: Code, Laufzeit, Systemtools, Systembibliotheken usw. Container werden erstellt aus eine schreibgeschützte Vorlage mit dem Namen Abbildung.

Images werden in der Regel aus einer Docker-Datei erstellt, einer Klartextdatei, die alle der Komponenten, die im Container enthalten sind. Diese Bilder werden dann und in einem Registrierung von denen sie heruntergeladen und führen Sie auf Ihrem Cluster aus. Weitere Informationen zur Containertechnologie finden Sie unter Docker-Grundlagen für Amazon ECS.

Aufgabendefinitionen

So bereiten Sie Ihre Anwendung für die Ausführung auf vor Amazon ECS, erstellen Sie eine Aufgabe Definition. Die Aufgabendefinition ist eine Textdatei im JSON-Format, die beschreibt einen oder mehrere Container bis zu maximal zehn, die Ihre Anwendung. Es kann als Vorlage für Ihre Anwendung betrachtet werden. Aufgabe -Definitionen geben verschiedene Parameter für Ihre Anwendung an. Beispiele für eine Aufgabe Definitionsparameter umfassen, welche Container verwendet werden sollen, welche Ports geöffnet werden sollen für Ihre Anwendung und welche Daten-Volumes mit den Containern in die Aufgabe. Die für die Aufgabendefinition verfügbaren spezifischen Parameter hängen von der Anforderungen Ihrer Anwendung. Weitere Informationen zum Erstellen von Aufgabendefinitionen finden Sie unter Amazon ECS Aufgabendefinitionen.

Es folgt ein Beispiel für eine Aufgabendefinition, die einen einzelnen Container enthält der einen NGINX-Webserver ausführt, der auf Fargate. Für ein besseres erweitertes Beispiel, das die Verwendung mehrerer Container in einer Aufgabendefinition demonstriert, siehe Beispiel für Aufgabendefinitionen. 

{
    "family": "webserver",
    "containerDefinitions": [
        {
            "name": "web",
            "image": "nginx",
            "memory": "100",
            "cpu": "99"
        },
    ],
    "requiresCompatibilities": [
        "FARGATE"
    ],
    "networkMode": "awsvpc",
    "memory": "512",
    "cpu": "256",
}

Aufgaben und Planung

Eine Aufgabe ist die Instanziierung einer Aufgabendefinition innerhalb eines -Cluster. Nachdem Sie eine Aufgabendefinition für Ihre Anwendung in erstellt haben Amazon ECS, können Sie die Anzahl der Aufgaben angeben, die auf Ihrem Cluster ausgeführt werden.

Die Schaltfläche Amazon ECS Der -Aufgaben-Scheduler ist für die Platzierung von Aufgaben in Ihrem Cluster verantwortlich. Es stehen mehrere verschiedene Planungsoptionen zur Verfügung. Sie können beispielsweise definieren Sie eine Wartung die eine bestimmte Anzahl von Aufgaben gleichzeitig ausführen. Für weitere Informationen über die verschiedenen Planungsoptionen verfügbar, siehe Planung Amazon ECS Aufgaben.

Clusters

und eine Amazon ECS bündeln ist eine logische Gruppierung von Aufgaben oder Services. Sie können einen oder mehrere Amazon EC2 -Instances, auch als -Container-Instances mit Ihrem Cluster, um Aufgaben auf auszuführen, oder können Sie die serverlose Infrastruktur verwenden, die Fargate stellt bereit. Wenn Ihre Aufgaben werden ausgeführt am Fargate, Ihre Cluster-Ressourcen werden verwaltet von Fargate.

Wann Sie zum ersten Mal Amazon ECS, wird ein Standard-Cluster für Sie erstellt, aber Sie können mehrere Cluster in einem Konto, um Ihre Ressourcen getrennt zu halten.

Weitere Informationen zum Erstellen von Clustern finden Sie unter Amazon ECS-Clustern. Weitere Informationen zum Starten des Containers -Instances und deren Registrierung bei Ihren Clustern finden Sie unter. Amazon ECS-Container-Instances.

Behälter-Agent

Die Schaltfläche Behälter-Agent wird auf jeder Container-Instance innerhalb einer ausgeführt Amazon ECS -Cluster. Es sendet Informationen über die aktuell ausgeführten Aufgaben der Ressource und Ressourcenauslastung, um Amazon ECS, und startet und stoppt Aufgaben, wenn sie eine Anfrage von Amazon ECS. Weitere Informationen finden Sie unter Amazon ECS-Container-Agent.

Erste Schritte mit Amazon ECS

Wenn Sie Amazon ECS zum ersten Mal, AWS Management Console für Amazon ECS bietet eine Assistent zum ersten Ausführen, der Sie durch die Definition einer Aufgabendefinition für einen Webserver führt, Konfigurieren eines Service und Starten Ihres ersten Fargate Aufgabe. Die Schaltfläche Der Assistent für die erste Ausführung wird dringend empfohlen für Benutzer, die noch keine Erfahrung mit Amazon ECS. Weitere Informationen finden Sie im Erste Schritte mit Amazon ECS unter Verwendung von Fargate Tutorial.

Alternativ können Sie die AWS Command Line Interface (AWS CLI) zu verwenden Amazon ECS. Für mehr Informationen, siehe Einrichten mit Amazon ECS.

Zugehörige Dienstleistungen

Amazon ECS kann zusammen mit den folgenden AWS Dienstleistungen:

AWS Identity and Access Management

IAM ist ein Webservice, mit dem Sie den Zugriff auf AWS sicher steuern können -Ressourcen für Ihre Benutzer. Verwenden IAM um zu steuern, wer Ihre AWS- Ressourcen (Authentifizierung) und welche Ressourcen sie auf welche Weise verwenden können (Autorisierung). Eingebucht Amazon ECS, , und Sie haben die Möglichkeit IAM kann verwendet werden, um den Zugriff auf die Container-Instance-Ebene mit IAM Rollen und auf Aufgabenebene mithilfe von IAM Aufgabenrollen. Weitere Informationen finden Sie unter Identitäts- und Zugriffsverwaltung für Amazon Elastic Container Service.

Amazon EC2 Auto Scaling

Auto Scaling ist ein Webservice, der es Ihnen ermöglicht, automatisch zu skalieren Ihre Aufgaben basierend auf benutzerdefinierten Richtlinien, Zustandsprüfungen und Zeitpläne. Sie können Auto Scaling mit einem Fargate Aufgabe innerhalb eines Service zur Skalierung als Reaktion auf eine Reihe von Metriken oder mit einem EC2 Aufgabe zum Skalieren der Container-Instances innerhalb Ihres Clusters. Weitere Informationen finden Sie unter Service Auto Scaling.

Elastic Load Balancing

Elastic Load Balancing verteilt eingehenden Anwendungsdatenverkehr automatisch über die Aufgaben in Ihrem Amazon ECS zu warten. Es ermöglicht Ihnen, ein höheres Maß an Fehlertoleranz in Ihren Anwendungen und bietet nahtlos die erforderliche Menge an Load-Balancing-Kapazität, die zur Verteilung des Anwendungsdatenverkehrs benötigt wird. Sie können Elastic Load Balancing um einen Endpunkt zu erstellen, der Datenverkehr über -Services in einem Cluster. Weitere Informationen finden Sie unter Lastausgleich von Diensten.

Amazon Elastic Container Registry

Amazon ECR ist eine verwaltete AWS Docker-Registrierungsservice, der sicher, skalierbar, und zuverlässig. Amazon ECR unterstützt private Docker-Repositorys mit ressourcenbasierten Berechtigungen mit IAM damit bestimmte Benutzer oder Aufgaben auf -Repositorys und -Images. Entwickler können die Docker-CLI verwenden, um und verwalten Sie Bilder. Weitere Informationen finden Sie im Amazon Elastic Container Registry-Benutzerhandbuch.

AWS CloudFormation

AWS CloudFormation bietet Entwicklern und Systemadministratoren eine einfache Möglichkeit, und eine Sammlung verwandter AWS Ressourcen, Bereitstellung und sie in einer geordneten und vorhersehbaren Weise zu aktualisieren. Sie können definieren, -Cluster, Aufgabendefinitionen und -Services als Entitäten in einem AWS CloudFormation -Skript. Für Weitere Informationen, siehe AWS CloudFormation-Vorlage Bezugsgröße.

Zugriff auf Amazon ECS

Sie können mit Amazon ECS wie folgt:

AWS Management Console

Die Schaltfläche AWS Management Console ist eine browserbasierte Schnittstelle zur Verwaltung Amazon ECS -Ressourcen. Die Schaltfläche -Konsole bietet eine visuelle Übersicht über den -Service. Sie können durch die Tutorials und schrittweise Anleitungen, während Sie neue Amazon ECS Funktionen ohne die zusätzliche Werkzeuge benötigen. Ein Tutorial, das Sie durch die Konsole, siehe Erste Schritte mit Amazon ECS.

AWS CloudFormation oder Terraform

AWS CloudFormation oder Terraform für Amazon ECS sind wirkungsvolle Wege, um Ihre Infrastruktur-als-Code. Sie ermöglichen es Ihnen, einfach zu verfolgen, welche Version von Ihre Vorlage oder AWS CloudFormation -Stapel jederzeit ausgeführt wird, und rollen Sie zu einem Vorgängerversion. Sie können Infrastruktur- und Anwendungsbereitstellungen durchführen in der gleichen automatisierten Art und Weise, AWS CloudFormation und Terraform gängige Formate, um Workloads für bereitstellen Amazon ECS von kontinuierlichen Lieferpipelines.

Weitere Informationen über AWS CloudFormation, siehe Erstellen Amazon ECS -Ressourcen mit AWS CloudFormation.

AWS Copilot Befehlszeilenschnittstelle

Die Schaltfläche AWS Copilot Die CLI ist eine Befehlszeilenschnittstelle, mit der Benutzer Anwendungen in Containern und Umgebungen auf Amazon ECS. Sie ist ein umfassendes Tool, das es einem Kunden ermöglicht, Anwendungen direkt von ihrem Quellcode, ohne dass sie verstehen müssen, AWS und Amazon ECS Grundstoffe wie Application Load Balancers, öffentliche Subnetze, Aufgaben, Services und Cluster. AWS Copilot erstellt AWS Ressourcen in Ihrem Namen von einem begutachteten Service Muster, z. B. ein lastausgeglichener Webdienst oder ein Backend-Dienst Bereitstellung einer sofort einsatzbereiten Produktionsumgebung für containerisierte Anwendungen. Sie können über eine AWS CodePipeline Pipeline über mehrere -Umgebungen, -Konten oder -Regionen, die alle innerhalb der CLI selbst verwaltet werden können. AWS Copilot ermöglicht es Ihnen, auch Bedieneraufgaben zu übernehmen, wie z. B. das Anzeigen von Protokollen und den Zustand Ihres -Dienstes.

AWS Copilot ist ein einziges Tool, mit dem sich Benutzer zuerst auf Anwendungen konzentrieren können. Weitere Informationen finden Sie unter Verwenden der AWS Copilot Befehlszeilenschnittstelle.

AWS CDK

Die Schaltfläche AWS Cloud Development Kit (AWS CDK) ist ein Open-Source-Software-Entwicklungs-Framework zum Modellieren und stellen Sie Ihre Cloud-Anwendungsressourcen mithilfe vertrauter Programmierung bereit Sprachen. AWS CDK stellt Ihre Ressourcen sicher und wiederholbar bereit bis zum AWS CloudFormation. Mit dem CDK können Benutzer ihre Umgebung mit weniger Codezeilen und verwenden dieselbe Sprache wie ihre Anwendungen. Amazon ECS stellt ein Modul auf dem CDK mit dem Namen ecs-patterns was zu einer gängige Architekturen. Ein verfügbares Muster ist ApplicationLoadBalancedFargateService() wodurch eine -Cluster, Aufgabendefinition und zusätzliche Ressourcen zum Ausführen eines Load Balancing Amazon ECS Dienst am AWS Fargate. Weitere Informationen finden Sie unter Erstellen eines Amazon ECS auf dem Bildschirm AWS Fargate Service mit der AWS CDK in der AWS Cloud Development Kit (AWS CDK)-Entwicklerhandbuch.

AWS CLI

Die Schaltfläche AWS Command Line Interface (AWS CLI) ist ein einheitliches Tool zur Verwaltung Ihrer AWS Dienstleistungen. Mit einem einzigen Tool zum Herunterladen und Konfigurieren können Sie mehrere AWS -Services über die Befehlszeile und automatisieren Sie über Skripts. Die Schaltfläche Amazon ECS Befehle in der AWS CLI sind ein Spiegelbild der Amazon ECS API. Verwenden der AWS CLI ist vorteilhaft für Benutzer, die ein Befehlszeilen-Tool und Skripting bevorzugen, aber auch wissen genau, welche Maßnahmen zu ergreifen sind, Amazon ECS oder für Benutzer, die mehr über die Amazon ECS APIs.

AWS bietet zwei Sätze von Befehlszeilen-Tools: die AWS Command Line Interface (AWS CLI) und die AWS-Tools für Windows PowerShell. Weitere Informationen finden Sie im Benutzerhandbuch für AWS Command Line Interface und die AWS-Tools für Windows PowerShell Benutzer Anleitung.

Amazon ECS Befehlszeilenschnittstelle

Sie können schnell Bereitstellen von Ressourcen, Push- und Pull-Images mithilfe von Amazon ECR, und überwachen Sie laufende Anwendungen auf Amazon ECS oder AWS Fargate. Sie können auch Container testen, die lokal zusammen mit Containern in der Cloud innerhalb der CLI. Für mehr Informationen, siehe Verwenden der Amazon ECS-Befehlszeilenschnittstelle.

Docker-CLI-Plugin für Amazon ECS

AWS und Docker haben zusammengearbeitet, um einen vereinfachten Entwickler Erfahrung, die es Ihnen ermöglicht, Container auf Amazon ECS direkt aus den Docker-Tools heraus. Sie können jetzt Ihre Container erstellen und testen lokal mithilfe von Docker Desktop und Docker Compose und stellen Sie dann auf Amazon ECS auf dem Bildschirm Fargate über die gleiche CLI.

Das Docker-CLI-Plugin für Amazon ECS ist derzeit in Beta. Für mehr Informationen, siehe Docker (Anhänger) CLI-Plugin für Amazon ECS auf GitHub.

Weitere Informationen finden Sie unter Amazon ECS Preisgestaltung.

Amazon ECS und Fargate bieten auch Savings Plans was erhebliche Einsparungen bei Ihren AWS Nutzung. Weitere Informationen finden Sie im Savings Plans Benutzerhandbuch.
